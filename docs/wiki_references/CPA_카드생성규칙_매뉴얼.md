# CPA 학습 자동화 시스템 — 카드 생성 및 트리거 규칙 상세 매뉴얼 (완성본)

이 문서는 마진노트(MarginNote)에 기록된 노드와 코멘트가 파이썬 엔진을 거쳐 **어떤 형태의 Anki 카드로 변환되는지**, 그 모든 디테일과 한계 조건, 자동화 패턴을 정의하는 궁극의 참조 문서입니다.

---

## 1. 카드에 들어갈 수 있는 데이터 형태 (지원 포맷)

마진노트의 코멘트에 어떤 형태의 데이터를 넣든 엔진은 완벽하게 Anki 앞/뒷면으로 매핑합니다. 텍스트, 이미지, 혼합 형태를 전부 지원합니다.

| 입력 데이터 형태 | Anki 카드에 반영되는 모습 | 비고 |
|----------------|------------------------|------|
| **텍스트만** | 텍스트 (Markdown 렌더링) | 줄바꿈, 볼드체 등 자동 적용 |
| **이미지만** | 이미지 원본 | 표, 강사 판서, 문제 캡처 등 |
| **텍스트 여러 개** | 작성된 순서대로 세로 나열 | 여러 개의 코멘트 블록 통합 |
| **이미지 여러 개** | 캡처된 순서대로 세로 나열 | 여러 장의 풀이 이미지 통합 |
| **텍스트 + 이미지 혼합** | 텍스트가 먼저, 이미지가 아래에 배치 | 가장 흔한 패턴 (문제 텍스트 + 풀이 캡처) |

---

## 2. 기본형 카드 (트리거가 없는 경우)

코멘트에 특별한 트리거 키워드(예: `주의:`)가 없는 경우, 일반적인 Q&A 카드로 취급됩니다.

- **앞면 (질문)**: 노드의 제목 (예: "2. 평가결과가 상반될 수 있는 두 투자안 비교")
- **뒷면 (답변)**: 노드의 내용 (코멘트 텍스트 및 첨부 이미지 전체)
- **복습 주기 (SRS)**: 표준 간격 (1분 → 10분 → 1일 ...)

### 💡 [매우 중요] 정의(Definition) 자동 감지 패턴
기본형 카드라도 내용 중에 **"개념 정의" 패턴**이 발견되면, 엔진은 기본 Q&A 카드 외에 **빈칸(Cloze) 카드 1장을 추가로 자동 생성**합니다.
엔진이 감지하는 정규식 패턴은 다음과 같습니다:
- `X이란 Y이다`
- `X은/는 Y을/를 말한다`
- `X은/는 Y을/를 의미한다`
- `X(이)라 함은 Y`
- `X: Y` (콜론 패턴)
*(결과: X 부분이 가려진 빈칸 카드가 자동 생성되어 무의식적인 용어 암기를 돕습니다)*

---

## 3. 특수 트리거 생성 규칙 (9대 키워드)

코멘트 첫 줄에 아래의 키워드를 적으면 엔진이 형태를 완전히 변환합니다.

| 트리거 키워드 | 작동 원리 및 카드 앞/뒷면 구조 | 타겟 복습 주기 |
|-------------|-----------------------------|--------------|
| **`주의:`** | **[함정/주의 카드]**<br>실수하기 쉬운 포인트를 묻습니다.<br>• 앞면: ⚠ [소단원명] 주의할 점은?<br>• 뒷면: 코멘트로 적은 텍스트 | 짧음 (빠른 반복) |
| **`강사:`** | **[강사 팁 카드]**<br>강사의 구두 설명이나 실무 팁을 묻습니다.<br>• 앞면: 🎤 [소단원명] 강사 보충 설명<br>• 뒷면: 코멘트로 적은 텍스트/판서 이미지 | 표준 (일반 간격) |
| **`절차:`** | **[점층적 빈칸 카드]**<br>1~4단계의 절차를 체화할 때 씁니다. 한 번에 다 묻지 않고 하나씩 가립니다.<br>• 카드 1: 1. [___]<br>• 카드 2: 1. A절차수행  2. [___]<br>• 카드 3: 1. A절차  2. B절차  3. [___] | 짧음 (단기 체화용) |
| **`공식:`** | **[공식 암기 카드]**<br>긴 수식의 각 요소를 하나씩 뚫어놓은 N장의 빈칸(Cloze) 카드를 무더기로 생성합니다. | 짧음 (단기 체화용) |
| **`가림:`** | **[Image Occlusion 카드]**<br>텍스트 없이 '비교표/T계정' 이미지만 있을 때 씁니다. Anki 내에서 빨간 사각형을 그려 그 밑의 글씨를 맞추게 됩니다. | 표준 (일반 간격) |
| **`분개:`** | **[분개 카드]**<br>회계처리를 묻습니다.<br>• 앞면: [상황 텍스트] 분개는?<br>• 뒷면: 차/대변 답안 (또는 분개 캡처 이미지) | 짧음 (단기 체화용) |
| **`빈출`** | **[간격 제한 타이머]**<br>새로운 카드를 만들지 않습니다. 대신 이 키워드가 붙은 노드와 그 산하에 있는 **모든 카드의 복습 주기 상한선을 무조건 60일로 강제 제한**합니다. 아무리 잘 맞춰도 두 달에 한 번은 무조건 봐야 합니다. | 최대 60일 제한 |

## 4. 메타인지 추적 및 연결 시스템 (2차 연습서 대응)

과거 수동으로 `연결:` 이나 `오답:` 이라 치고 텍스트를 적던 방식은 **완전히 폐기**되었습니다. 이제 진정한 '구조'와 '통계'가 만납니다.

### 4.1. 다중 위계 문제의 통합 (Logseq 연동)
2차 연습서는 **[문제 1번 (자료)]** 아래에 **[물음 1, 물음 2]**가 달립니다.
- 엔진은 이를 Logseq에서 볼 수 있는 "계층형 아웃라이너" 마크다운으로 추출합니다.
- Anki에서는 `물음 2` 카드가 단독으로 튀어나옵니다.
- ❗이때 문제의 맥락(공통 자료)이 기억 안 나면, 카드 하단의 **`[ L ]` (Logseq 딥링크)** 아이콘을 누릅니다. 즉시 Logseq이 열리며 해당 문제의 전체 위계 구조를 한눈에 보여줍니다.

### 4.2. 오답 원인망 형성 (MarginNote 선 긋기)
틀린 원인을 텍스트로 치지 마십시오. 시각적 연결이 전부입니다.
- **상황**: 연습서 [물음 2]를 풀다가, "아 이거 '자산손상'과 '법인세' 개념을 몰라서 틀렸구나" 깨달았습니다.
- **행동**: 마진노트를 열고, `자산손상` 개념 노드와 `법인세` 개념 노드를 마우스로 쓱 끌어당겨 [물음 2] 노드에 **점선(Link)으로 이어주기만** 합니다.
- **결과**: 엔진이 `ZNOTES_TEXT`에 몰래 새겨진 `marginnote4app://...` 연결 데이터를 자동 파싱하여, 이 문제의 원인이 무엇인지 마크다운과 Anki의 Hidden Tag에 `#약점::자산손상`, `#약점::법인세`로 영구 박아넣습니다.

### 4.3. 궁극의 오답 통계 (Anki Revlog + Obsidian Dataview)
사용자가 직접 "나 이거 5번 틀렸음" 이라고 적을 필요가 없습니다. 진실은 데이터에 있습니다.
- **Anki의 통계 (진실의 방)**: 사용자님이 Anki에서 카드를 풀고 `[다시(Again)]` 버튼을 누른 누적 횟수(Lapses)를 Python 엔진이 Anki SQLite DB에서 직접 긁어옵니다.
- **Obsidian 대시보드 (통합 시각화)**: 
  엔진은 (1) Anki에서 긁어온 실제 오답 횟수와 (2) MarginNote에서 그어놓은 원인 개념망을 결합하여 옵시디언 마크다운으로 출력합니다.
  결과적으로 옵시디언 대시보드에는 다음과 같은 랭킹이 실시간으로 뜹니다.
  > 📊 **나의 2차 연습서 약점 랭킹**
  > 1. `[[8장 자산손상]]` : 관련 문제 도합 27번 틀림 (치명적 약점)
  > 2. `[[12장 법인세]]` : 관련 문제 도합 15번 틀림
  > 3. `[[5장 리스]]` : 관련 문제 도합 3번 틀림
