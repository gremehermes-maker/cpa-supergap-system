# CPA 학습 자동화 시스템 — 예상 문제점 및 궁극의 개선안 (최종본)

본 문서는 CPA 마스터플랜 시스템의 아키텍처에서 발생할 수 있는 모든 한계점과, 헤르메스 에이전트 및 마진노트4의 고유 URL 스킴을 활용한 **가장 진보된 형태의 최종 해결책**을 통합한 문서입니다.

---

## 1. 정규표현식 파서(Regex) vs 헤르메스 에이전트 파싱
* **예상 문제점:** 파이썬 정규표현식(`주의:`, `절차:` 등)은 띄어쓰기나 미세한 오타에 파싱이 고장 날 위험이 있습니다.
* **최종 개선안 (사용자 역량 기반 하이브리드):**
  * 사용자가 오타 없이 시스템 룰에 맞게 정확히 트리거를 작성한다면 기존 파이썬 스크립트만으로 비용 없이 100% 완벽하게 카드가 생성됩니다.
  * 단, 모바일에서 대충 메모했거나 비정형 데이터(오타 섞인 문장 등)를 파싱해야 할 때만 헤르메스 에이전트(LLM)에게 데이터를 넘겨 교정 및 JSON 규격화를 맡기는 선택적 하이브리드 방식을 적용합니다.

## 2. 2차 연습서 오답의 '약점망 연결' (스파게티 링크 종식)
* **예상 문제점:** 1단원 개념 노드와 15단원의 문제 노드를 마인드맵 상에서 마우스로 드래그해서 잇는 것은 노드가 많아지면 물리적으로 불가능합니다. (Spaghetti Linking)
* **최종 개선안 (헤르메스 TOC 자동 매핑):**
  * **[원리]** 마진노트4의 각 노드는 고유 URL(`marginnote4app://...`)을 갖습니다. 사용자가 직접 URL을 딸 필요가 없습니다.
  * **[1단계: 인덱싱]** Phase 0(교재 분해) 단계에서, 파이썬 스크립트가 교재 목차 번호(예: `1.1.1 자산손상`)와 마진노트의 고유 URL을 1:1로 매핑하여 'TOC-URL 인덱스 DB'를 구축합니다.
  * **[2단계: 명령]** 사용자는 오답 발생 시, 헤르메스 에이전트에게 챗/슬랙으로 **"1.1.1 목차랑 1.3.2 연습서 문제 좀 연결해줘"**라고만 지시합니다.
  * **[3단계: 자동 연결]** 헤르메스 에이전트가 인덱스 DB에서 두 목차의 고유 URL을 뽑아내어, 내부 마크다운 파일(혹은 DB)에 텍스트 형태의 링크로 쏴줍니다. 손가락 하나 까딱하지 않고 완벽한 연결망이 구축됩니다.

## 3. 다중 위계 문제 파싱 보호 (Logseq 들여쓰기 활용)
* **예상 문제점:** 2차 연습서의 복잡한 구조(`공통자료 → 물음1 자료 → 물음1`)가 마진노트에서 추출되어 Anki로 직행하면 위계가 무너져 맥락을 상실합니다.
* **최종 개선안 (마크다운 SSOT 구조):** 파싱 순서를 `마진노트 → 마크다운(Logseq) → Anki`로 변경합니다.
  * **[예시로 보는 작동 원리]**
  * 1. 마진노트에서 무작위로 추출된 날것의 텍스트:
    ```text
    [문제 1 공통 자료]
    물음 1. (가) 계산하시오.
    물음 2. 분개하시오.
    ```
  * 2. 사용자가 Logseq을 열어 **탭(Tab)** 키로 들여쓰기만 맞춰줍니다. (시각적/직관적)
    ```markdown
    - [문제 1 공통 자료]
      - 물음 1. (가) 계산하시오.
      - 물음 2. 분개하시오.
    ```
  * 3. Anki 자동화 스크립트는 이 **Logseq의 마크다운 들여쓰기(Indentation)**를 읽습니다. "아, 물음1/2는 한 칸 들어가 있으니 부모가 [문제 1]이구나!"라고 인식하여, 물음1 카드를 만들 때 뒷면에 [문제 1 공통 자료]의 내용을 자동으로 때려 박아줍니다.

## 4. 미디어 저장 공간 폭발 및 모바일 동기화 지연
* **예상 문제점:** 수만 장의 교재/판서 이미지로 인해 AnkiWeb 동기화가 터지거나, 모바일(Galaxy) 동기화 시 엄청난 지연과 배터리 광탈이 발생합니다.
* **최종 개선안 (압축 및 헤르메스 스마트 푸시):**
  * **[1차: 자동 압축]** 이미지가 Anki로 넘어가기 직전 모바일 친화적 해상도 축소 및 `.webp` 포맷으로 강제 변환하여 용량을 80% 줄입니다.
  * **[2차: 데일리 덱(Daily Deck) 시스템]** 나중에 데이터가 극도로 방대해지면, 헤르메스 에이전트가 매일 아침 FSRS 스케줄러를 분석하여 **"오늘 복습해야 할 카드"**만 추출한 아주 가벼운 미니 덱(Mini Deck)을 생성, 모바일 폰으로 쏴주는(Push) 방식으로 발전시킵니다. 무거운 전체 동기화가 필요 없어집니다.

## 5. 헤르메스 에이전트의 환각 방지 (이기종 교차 검증)
* **예상 문제점:** 튜터와 체커 에이전트가 같은 AI 뇌를 쓰면 같은 논리에서 똑같이 환각(Hallucination)을 겪어 틀린 답을 맞다고 승인해 버릴 수 있습니다.
* **최종 개선안 (OpenCode Go 활용):**
  * 사용 중인 **OpenCode Go** 구독의 장점을 극대화합니다.
  * 오픈코드 고는 API 하나로 여러 회사의 최신 모델을 스위칭할 수 있습니다. 튜터 에이전트에는 논리 추론이 강한 `Claude 3.5 Sonnet`을 할당하고, 체커 에이전트에는 팩트체크가 강한 `Qwen-Max` 또는 `Gemini 1.5 Pro`를 할당합니다.
  * 서로 출신(아키텍처)이 다른 두 뇌가 서로를 감시(Cross-check)하게 만들어 회계/세법 로직의 환각을 완벽에 가깝게 방어합니다.
