# CPA 학습 자동화 시스템 — [Phase 5] LLM OS 및 멀티 에이전트 확장 아키텍처

> **Reference:** 이 문서는 `CPA_마스터플랜_최종본.md`의 후속 확장판입니다. 원본 시스템(Phase 0~4) 위에 자율형 AI 에이전트를 결합하여 완벽한 '인공지능 수험 컨설턴트' 환경을 구축하는 내용을 담고 있습니다.

## 1. 안드레 카파시(Andrej Karpathy) 'LLM OS' 기반 메모리 아키텍처

LLM을 단순한 챗봇이 아닌 **'운영체제의 CPU'**로, 컨텍스트 윈도우를 **'RAM'**으로, Obsidian 기반의 LLM Wiki를 **'하드디스크(Disk)'**로 취급하는 아키텍처를 도입합니다. 일회성 세션의 한계를 극복하고 사용자의 학습 상태를 영구적으로 기억합니다.

### A. 컨텍스트 윈도우(RAM) 관리의 핵심: 전담 [Memory Editor] 에이전트
* 토큰 한계를 극복하기 위해 역할을 극단적으로 쪼갭니다.
* **Memory Editor (요약/추출 전담 에이전트):** 이 에이전트의 유일한 임무는 현재 진행 중인 세션(RAM)의 수많은 대화와 분석 내용 중 **'정수(Core Insight)'만을 추출하는 것**입니다. 질문에 답하거나 스케줄을 짜지 않습니다. 오직 세션이 끝날 때마다 핵심만 요약해서 LLM Wiki(Disk)에 **'던져넣고(Write)'** 사라집니다.

### B. 살아 숨 쉬는 LLM Wiki (Persistent Memory)
* 일회성 컨텍스트 창은 휘발되지만, Memory Editor 에이전트가 지속적으로 요약해 던진 정보들은 Obsidian 내의 `[LLM_Wiki]` 폴더에 영구적으로 축적됩니다.
* **작동 방식:** 튜터나 스케줄러 에이전트는 사용자와 대화하기 전(CPU 프로세스 시작 전), 반드시 LLM Wiki(Disk)의 핵심 페이지를 Context Window(RAM)로 **'로드(Read)'**합니다.
* *효과:* 수개월 전의 약점, 사용자의 학습 습관 등을 단 1KB의 요약 토큰만으로 완벽하게 기억하고 소통할 수 있습니다.

## 2. 특화 에이전트의 역할 분담 (Slack 기반 협업 통신)

기존에 구축된 Docker 및 Slack 환경을 활용하여, 환각(Hallucination)을 서로 감시하고 협업하는 다중 에이전트 시스템을 구성합니다.

1. **👨‍🏫 Tutor Agent (CPU 1 - 지식 처리):** 
   - 5TB 구글 드라이브에 구축된 `.txt/.srt` 강의 스크립트 기반 RAG 검색
   - 마진노트 연결망(오답 링크) 및 웹 검색을 융합하여 사용자의 재무관리/회계학 질문에 심층 답변.
2. **🕵️‍♂️ Scheduler Agent (CPU 2 - 행동 처리):** 
   - Anki SQLite와 상호작용하여 진도 지연 상황 발생 시 듀데이트를 물리적으로 재조정(Rescheduling).
3. **✍️ Memory Editor Agent (Memory Controller):** 
   - 슬랙의 대화 흐름을 모니터링하다가 가치 있는 사용자 약점/오답 원인만 추출하여 LLM Wiki 업데이트.
4. **🛡️ Checker Agent (환각 방지 필터):** 
   - 튜터 에이전트가 내놓은 답이 IFRS 기준서, 교재 본문, 혹은 LLM Wiki 내용과 충돌하지 않는지 최종 검수(Cross-check).

## 3. 에이전트 안전망 및 5TB 무제한 스냅샷 백업

사용자의 5TB 구글 드라이브 인프라를 적극 활용하여, 용량 제한 없는 완벽한 복구 환경과 에이전트 통제 프로토콜을 설정합니다.

### A. 무제한 스냅샷 백업 (Unlimited Archiving)
* 에이전트가 Anki DB를 수정하거나 LLM Wiki를 업데이트하기 직전, `YYYYMMDD_HHMMSS.zip` 형태로 마크다운과 SQLite DB 전체를 압축하여 구글 드라이브에 스냅샷을 남깁니다.
* 용량 제한이 없으므로 삭제 정책 없이 무한정 보존하여 언제든 특정 시점으로 롤백(Roll-back)이 가능합니다.

### B. 하이브리드 결재 시스템 (Confidence Threshold UX)
에이전트의 권한 남용을 막기 위한 행동 강령입니다.
* **Low Impact (자동 반영):** "어제 미접속으로 인한 1일 연기", "LLM Wiki 요약 기록" 등 비파괴적 작업은 자동 수행 후 Slack 통보.
* **High Impact (승인 요청):** "특정 단원 500개 카드 강제 Suspend" 등 전략적 판단이나 대규모 수정은 Slack에 승인 버튼(Approve/Reject)을 송출하여 사용자의 허락을 득한 후 실행.
