# CPA 학습 자동화 시스템 — 개발팀 WBS 및 아키텍처 설계서 (최종판)

> **💡 [문서 개요]**
> 본 문서는 시스템의 각 모듈(MarginNote, Python, Logseq/Obsidian, Anki, AI Agent)이 독립적으로 포괄하는 기술 스펙과 개발 범위를 선언하고, 이들이 최종적으로 어떻게 파이프라인으로 결합되는지를 정의하는 **팀 업무 분장(WBS)용 아키텍처 설계서**입니다.
> 
> 특히 **모든 하위 구조도 항목에 고유 식별 번호(Task ID)**를 부여하여 커뮤니케이션의 오해를 없애고 철저한 정보 관리가 가능하도록 고안되었습니다. 모든 팀원은 아래 부여된 `[1.1]`, `[2.3]` 등의 번호로 태스크를 소통해야 합니다.

### 🛠️ 컴포넌트별 세부 스펙 및 히스토리 명세 (WBS Annotations)

과거 시스템의 시행착오와 설계 철학이 모두 녹아있는 15개 하위 항목의 상세 주석(Annotation)입니다. 개발자는 기능 명세와 그 이면의 의도를 모두 이해해야 합니다.

#### [1] MarginNote Team (데이터 창고 및 1차 UI 제어)
* **[1.1] 3대 교재 뼈대 해체 자동화**
  * **기능:** 교재의 PDF를 통째로 읽는 것이 아니라, 용도에 맞게 정밀하게 찢어 마진노트의 초기 뼈대로 Import 합니다.
  * **💡 히스토리 (의도):** 단순 파싱이 아닙니다. Slicer/OPML Generator를 통해 기본서는 향후 개정판에 대비하여 목차(`TOC_Mapping.json`)만 별도 분리합니다. 1차 객관식은 [문제+해설] 단위로 묶어 자르고, 2차 연습서는 매우 복잡한 [공통자료 → 물음1,2 → 해설] 다중 위계로 정밀하게 크롭하여 수험생의 인지 과부하를 줄입니다.
* **[1.2] ZCOLOR 색상 매핑 룰**
  * **기능:** 노드에 칠한 빨강(2), 초록(3), 파랑(4) 색상 값만이 DB에 저장되어 카드의 성격을 결정합니다.
  * **💡 히스토리 (의도):** 과거 수험생이 코멘트 창에 `주의:`, `절차:` 등을 수동으로 일일이 타이핑하며 에너지를 낭비하던 주객전도 현상을 전면 폐기했습니다. 이제는 펜슬 터치(색상) 한 번이 완전 자동화 트리거가 됩니다.
* **[1.3] ZGROUPNOTEID 위계 트리 구축**
  * **기능:** 사용자가 마진노트에서 노드를 선으로 묶는 순간 이 아이디가 형성되며, 이것이 시스템의 다중 위계 단일 진실(Source of Truth)이 됩니다.
  * **💡 히스토리 (의도):** 구버전에서는 사용자가 Logseq에서 직접 탭(Tab) 키를 쳐서 위계를 잡으려다 스페이스 한 칸 실수로 구조 전체가 와해되는 치명적 한계가 있었습니다. 이를 방어하기 위해 오직 마진노트의 마인드맵 연결선만이 들여쓰기를 통제하도록 아키텍처를 뒤집었습니다.

#### [2] Python Team (동기화 엔진 및 백엔드 코어)
* **[2.1] PRAGMA 스키마 무결성 검증기**
  * **기능:** 파이썬이 마진노트 DB를 읽기 전, `PRAGMA table_info`를 실행해 필수 컬럼 존재 여부를 1차 검사합니다.
  * **💡 히스토리 (의도):** 비공식 DB 파싱의 치명적 약점(앱 메이저 업데이트 시 테이블 변경으로 전체 데이터가 박살나는 현상)을 방어하기 위한 최전선 안전망입니다. 에러 시 즉각 슬랙 봇 알람이 송출됩니다.
* **[2.2] Frontmatter 마크다운 파서**
  * **기능:** 마진노트의 SQLite를 파싱하여 Logseq과 Obsidian이 인식할 수 있는 메타데이터(Frontmatter) 포맷의 마크다운 파일로 동적 생성합니다.
* **[2.3] AnkiConnect 딥링크 브릿지 [M/L/O]**
  * **기능:** 생성되는 모든 Anki 카드 뒷면에 `[ M ]` (마진노트 원문 URL), `[ L ]` (Logseq 위계 맥락), `[ O ]` (Obsidian 지식그래프) 딥링크 버튼을 무조건 주입합니다.
  * **💡 히스토리 (의도):** 안키의 가장 큰 단점인 '파편화(단편적인 카드 때문에 문맥을 잊어버림)'를 막기 위한 핵심 브릿지입니다. 안키에서 복습하다가 이해가 안 되면 딥링크를 눌러 언제든 전체 숲(맥락)으로 순간이동(Reverse-Lookup)할 수 있습니다.
* **[2.4] 5TB 원자적 백업 스크립트**
  * **기능:** 마진노트 원본, 옵시디언/로그시크 볼트, 안키 DB를 한 날 한 시에 묶어 단일 ZIP 파일로 구글 드라이브(5TB)에 압축 보관합니다.
  * **💡 히스토리 (의도):** 과거 특정 DB(예: 안키만 롤백)만 롤백했을 때 고아 데이터(Orphan)가 증식하여 시스템이 완전히 꼬였던 '시점 불일치(Time-Sync Mismatch)' 사태를 원천 차단하기 위한 통째 덮어쓰기 백업망입니다.

#### [3] Logseq & Obsidian Team (계층/메타인지 프론트엔드)
* **[3.1] Logseq: 모든 카드 맥락 관리 아웃라이너**
  * **기능:** 파이썬이 던져주는 데이터를 받아 모든 카드의 부모-자식 위계 트리를 직관적인 아웃라이너 형태로 랜더링합니다.
  * **💡 히스토리 (의도):** 단순 2차 연습서 뷰어가 아닙니다. 안키에서 `[ L ]` 딥링크를 눌렀을 때 사용자를 맞이하여, 파편화된 카드가 전체 교재 숲에서 어디에 위치하는지 즉각적인 **'전역 맥락 관리(Context Management)'**를 제공하는 핵심 공간입니다.
* **[3.2] Obsidian: 에이전트 LLM Wiki 저장소**
  * **기능:** 자율 에이전트(Hermes 등)가 사용자의 약점과 조건들을 구조화(JSON/YAML)하여 지속적으로 읽고 쓰는 영구 메모리 볼트 역할을 수행합니다.
  * **💡 히스토리 (의도):** Karpathy OS 철학의 디스크(Disk)에 해당합니다. 단순한 마크다운 뷰어를 넘어, LLM 에이전트가 사용자의 인지 상태를 기록하고 RAG 검색을 수행하는 백엔드 지식망입니다.
* **[3.3] Obsidian: Dataview 약점 대시보드**
  * **기능:** 안키 내부의 Revlog 테이블(사용자가 '다시(Again)'를 누른 횟수)을 끌어와 연결망 태그와 결합하여 실시간으로 랭킹 보드를 시각화합니다.
  * **💡 히스토리 (의도):** 수험생이 "내가 지금 어느 단원의 어떤 개념 때문에 시험에서 자꾸 틀리는지(예: 1위 자산손상 27회 오답)" 메타인지를 완벽히 통제할 수 있게 해주는 궁극의 관제탑입니다.

#### [4] Anki System Team (복습 및 스케줄링)
* **[4.1] Image Occlusion 가림 워크플로우**
  * **기능:** 파이썬은 빈 이미지 뼈대만 전송하고, 사용자는 강의 당일 Anki Desktop에서 플러그인(Image Occlusion Enhanced)으로 빨간 박스를 직접 치는 것을 정규 스케줄화합니다.
  * **💡 히스토리 (의도):** 파이썬이 화면의 (X, Y) 좌표를 알아서 가려주는 컴퓨터 비전 오버엔지니어링을 포기하고 도출된 가장 현실적이고 안정적인 타협안입니다.
* **[4.2] FSRS 망각곡선 최적화**
  * **기능:** 쪼개진 덱 없이 오직 단일 덱 체제하에서 FSRS 알고리즘이 복습 주기를 수학적으로 최적화하여 100% 통제합니다. (구시대적 SM-2 알고리즘 폐기).
* **[4.3] 안드로이드 단방향 읽기 전용 락(Lock)**
  * **기능:** 모바일(AnkiDroid) 환경에서는 어떠한 이유로든 카드의 내용을 수정하지 못하게 룰로 막고, 오직 '리뷰(읽기)'만 수행하여 통계만 누적시킵니다.
  * **💡 히스토리 (의도):** 이동 중 폰에서 수정된 내용이 PC와 동기화될 때 덮어쓰기 논리 충돌로 날아가는 아키텍처적 결함을 막기 위한 강력한 단방향 안전장치입니다.

#### [5] AI Agent Team (LLM 지식 베이스 및 봇 제어)
* **[5.1] STT 오염 정제용 Custom Dict**
  * **기능:** 강의 녹음(STT)이 지식베이스(ChromaDB)에 들어가기 전, 마진노트에서 자동 추출된 '사용자 전용 용어 사전'과 소형 LLM을 거치게 하여 강제로 정제합니다.
  * **💡 히스토리 (의도):** RAG의 고질적 문제인 환각(Data Poisoning) 방어벽입니다. "자본유지 젓갈법" 같이 심각한 오탈자가 유입되면 봇이 엉뚱한 대답을 하는 것을 막기 위해 사전에 "자본유지 접근법"으로 무조건 치환합니다.
* **[5.2] Vector Search 목차 추천 봇**
  * **기능:** 슬랙으로 틀린 연습서 문제의 텍스트를 던져주면, 봇이 교재 목차(`TOC_Mapping.json`)를 벡터 검색하여 맵핑 후보를 추천해줍니다.
  * **💡 히스토리 (의도):** 사용자가 과거처럼 목차 번호(`1.3.2`)를 달달 외워서 수동으로 엮어주던 극한의 인지적 과부하를 기술로 제거한 원클릭 승인 시스템입니다.
* **[5.3] FSRS Postpone API 통제 모듈**
  * **기능:** 에이전트는 절대 안키 SQLite의 원본 `due` 날짜를 임의 조작하지 못하며, 오직 정식 `FSRS Postpone API`만을 호출하도록 권한이 락다운(Lockdown)됩니다.
  * **💡 히스토리 (의도):** LLM이 마음대로 복습 일정을 미뤘다가 간격 알고리즘이 붕괴되어 카드 더미가 폭발(Interval Explosion)하는 사태를 막기 위한 핵심 수학적 방어 로직입니다.

## 2. 통합 엔드투엔드(E2E) 파이프라인

개별 번호가 부여된 하위 모듈들이 최종 사용자 환경에서 어떤 순서로 결합되어 순환 고리(Circular Workflow)를 만들어내는지 보여주는 구조도입니다. 업무 소통 시 "2단계 마진노트의 [1.2] 태스크와 5단계 안키의 [4.3] 룰이 충돌하지 않는지" 확인하는 식으로 커뮤니케이션합니다.

```mermaid
graph TD
    %% [1] Input & Pre-processing
    subgraph S1 ["1단계. 지식 & 뼈대 파이프라인"]
        A1[기본서 PDF] -->|TOC_Mapping.json 추출 [1.1]| B[MarginNote 4 뼈대 Import]
        A2[1차 객관식 문제집] -->|문항/해설 슬라이싱 [1.1]| B
        A3[2차 연습서 문제집] -->|공통자료-물음 다중 위계 구조화 [1.1]| B
        S[강의 STT 녹음] -->|Custom Dict 정제 [5.1]| RAG[(ChromaDB 지식베이스)]
    end

    %% [2] User Interaction (MarginNote)
    subgraph S2 ["2단계. MarginNote4 (창고 & 1차 제어)"]
        B --> MN_Study[사용자 학습 및 트리거 지정]
        MN_Study -->|빨강/파랑/초록 색칠| MN_Color[ZCOLOR: 자동 트리거 [1.2]]

        MN_Color --> DB[(.marginnotes SQLite)]
        MN_Group --> DB
    end

    %% [3] Python Parser
    subgraph S3 ["3단계. Python 동기화 엔진"]
        DB -->|PRAGMA 무결성 검증 [2.1]| Py1[DB 파싱 및 마크다운 변환기 [2.2]]
        Py1 -->|모든 카드 맥락 주입| MD_Logseq[Logseq Vault]
        Py1 -->|LLM Wiki 데이터 주입| MD_Obs[Obsidian Vault]
        Py1 --> Py2[AnkiConnect 딥링크 브릿지 주입 [2.3]]
    end

    %% [4] Logseq & Obsidian
    subgraph S4 ["4단계. Logseq & Obsidian (계층/메타인지 관제)"]
        MD_Logseq --> Logseq[Logseq: 모든 카드 맥락 관리 아웃라이너 [3.1]]
        MD_Obs --> Obs[Obsidian: 에이전트 LLM Wiki 및 대시보드 [3.2, 3.3]]
    end

    %% [5] Anki
    subgraph S5 ["5단계. Anki 복습 시스템"]
        Py2 -->|포트 8765 Health Check| Anki_App[Anki Desktop]
        Anki_App -.->|딥링크 [L] / [O] 로 맥락 복원 순간이동| MD_Logseq
        Anki_App -.->|딥링크 [M] 으로 원본 PDF 순간이동| MN_Study
        Anki_App -->|당일 수동 작업| IO[가림 뼈대카드 수동 박스 처리 [4.1]]
        Anki_App -->|모바일 동기화| Mobile[AnkiDroid: 단방향 읽기 전용 [4.3]]
        Mobile -->|다시(Again) 통계| Revlog[(Revlog 오답 통계 DB [4.2])]
        Revlog --> Obs
    end

    %% [6] AI Agents
    subgraph S6 ["6단계. AI 에이전트 & Slack"]
        Obs -.-> Hermes[Hermes 에이전트]
        RAG -.-> Hermes
        Hermes -->|Vector Search 추천 봇 [5.2]| Slack[Slack/Telegram 봇]
        Slack -->|원클릭 승인| MD_Obs
        Slack -->|FSRS Postpone API 전용 호출 [5.3]| Anki_App
    end

    %% [7] Backup
    subgraph S7 ["7단계. 5TB 원자적 백업망"]
        DB -.->|한 날 한 시 단일 ZIP 압축 [2.4]| Backup[(Google Drive 백업)]
        MD_Logseq -.-> Backup
        MD_Obs -.-> Backup
        Anki_App -.-> Backup
    end
```
