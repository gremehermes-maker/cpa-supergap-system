# CPA 학습 자동화 시스템 — 그랜드 마스터플랜 통합본 (Grand Masterplan)

> **💡 [문서 개요]**
> 본 문서는 처음 시스템을 접하는 사람도 전체 구조와 디테일을 완벽히 이해할 수 있도록, 흩어져 있던 **모든 마스터플랜, 하위 연동 매뉴얼(통합, 카드, 생성규칙), AI 에이전트 확장팩, 그리고 최신 시스템 개선안(v3.0)**까지 단 하나의 내용 유실 없이 집대성한 **궁극의 단일 문서(Single Source of Truth)**입니다.

---

## 1. 4대 코어 아키텍처와 데이터 파이프라인

데이터는 '구조 뼈대 생성'부터 '약점 역추적'까지 완벽한 순환 고리(Circular Loop)를 형성합니다. 학습 과정에서 발생하는 모든 데이터는 하나의 원천(MarginNote)에서 출발해 각 앱의 특성에 맞게 분리 및 통합됩니다.

| 시스템 | 역할 (기능) | 상세 설명 |
|--------|-----------|---------|
| **MarginNote4** | **창고 & 입력단** | 교재, 필기, 오답이 모이는 원천 DB. PDF 발췌, 트리거 코멘트 작성. 각 노드는 고유 URL(`marginnote4app://`)을 가짐. |
| **Logseq** | **계층 통제실** | 2차 연습서의 복잡한 [공통자료]-[물음1,2] 위계를 마크다운 **들여쓰기(Tab)**로 직관적으로 통제하는 아웃라이너. |
| **Obsidian** | **메타인지 대시보드** | Anki 오답 통계와 원인 연결망을 취합해 "내가 어떤 개념 때문에 몇 번 틀렸는지" 실시간 약점 랭킹 제공. |
| **Anki** | **자동 복습기** | 쪼개진 덱 없이 FSRS 망각곡선이 단일 덱에서 100% 스케줄 통제. 딥링크 버튼으로 원문/맥락으로 이동. |

## 2. Phase 별 시스템 설계 및 실행 계획

시스템의 구현과 작동은 크게 7단계(Phase 0~6)로 나뉩니다.

### [Phase 0] 교재 분해 파이프라인 (TOC 자동 매핑)
기본서와 문제집을 통째로 보지 않고 뼈대 단위로 해체합니다.
* **실행 스크립트:** 기본서 북마크 파싱(OPML 생성), 객관식/연습서 문항 정밀 크롭(Slicer).
* **✨ [최신 개선] 목차(TOC)-URL 인덱스 DB 구축:** 마진노트로 임포트된 목차(예: `1.1.1 자산손상`)와 마진노트의 고유 URL을 파이썬이 미리 1:1로 매핑하여 DB로 만듭니다. 향후 오답 연결 시 사용자가 URL을 복사할 필요 없이 챗봇에게 번호만 불러주면 되게끔 셋업합니다.

### [Phase 1~2] 마진노트 DB 추출 및 마크다운 변환
마진노트의 로컬 DB(SQLite)를 읽어내어 파일 기반 마크다운(Markdown)으로 변환합니다.
* **✨ [최신 개선] Snapshot 미러링:** 앱 크래시 방지를 위해 파이썬은 라이브 DB가 아닌 `/tmp/` 폴더로 복사된 스냅샷 DB만 읽습니다.
* **✨ [최신 개선] 유연한 위계 파싱 (마크다운 들여쓰기 룰):**
  마진노트의 엉성한 그룹핑을 그대로 Anki로 보내지 않습니다. 
  1) 마진노트 데이터가 Logseq으로 먼저 넘어옵니다. 
  2) 사용자가 Logseq에서 탭(Tab) 키로 `공통자료` 하위에 `물음1, 물음2`를 예쁘게 들여쓰기합니다. 
  3) 파이썬 스크립트가 이 **마크다운의 들여쓰기를 읽어 부모-자식 맥락을 완벽히 매핑**한 뒤 Anki로 보냅니다.
* **✨ [최신 개선] WebP 자동 압축:** 이미지 복사 전 해상도를 축소하고 `.webp` 포맷으로 변환해 모바일 동기화 지연과 용량 폭발을 80% 줄입니다.

### [Phase 3~4] Anki 카드 생성과 딥링크 브릿지
* **✨ [최신 개선] Regex vs LLM 파서 하이브리드:** 사용자가 트리거(예: `주의:`)를 오타 없이 입력하면 파이썬 정규식(Regex)으로 공짜로 카드를 생성합니다. 하지만 모바일에서 대충 쓴 오타나 비정형 데이터는 헤르메스 에이전트(LLM)에게 보내 알아서 JSON으로 파싱하게 합니다.
* **단방향 덮어쓰기 방어 (Inbox 룰):** 모바일(Galaxy) Anki에서 카드를 직접 수정하면 동기화 시 덮어씌워져 증발합니다. 모바일에서는 '조회/복습'만 하고, 수정할 내용은 슬랙/텔레그램 봇(Inbox)에 보내 Mac에서 일괄 처리합니다.
* **딥링크 삽입:** 카드 뒷면에 `[ M ]` (마진노트 원문), `[ L ]` (Logseq 위계 맥락), `[ O ]` (Obsidian 지식그래프) 딥링크를 심어 언제든 숲으로 순간이동합니다.

### [Phase 5] 카파시(Karpathy) LLM OS와 에이전트 확장 (Hermes)
LLM을 'CPU'로, Obsidian 폴더를 '하드디스크(Disk)'로 쓰는 다중 에이전트 환경입니다.
* **Memory Editor (요약/추출 담당):** 휘발성 대화를 한 줄로 요약해 `LLM_Wiki/User_State.md`에 기록합니다. **(✨ [최신 개선] 원본 대화 로그는 HTML `<details>` 태그 안에 숨겨 보존합니다.)**
* **Tutor & Checker (지식과 검증):** **✨ [최신 개선] OpenCode Go를 통한 이기종 교차 검증.** 튜터는 논리력이 뛰어난 `Claude 3.5 Sonnet`을 쓰고, 체커는 지식 베이스가 탄탄한 `Qwen/Gemini`를 씁니다. 서로 다른 뇌가 상호 감시하여 회계/세법 로직의 환각(Hallucination)을 원천 차단하고, 체커는 반드시 IFRS 문단 번호를 색인합니다.
* **Scheduler (행동 조정 담당):** **✨ [최신 개선] 미니 데일리 덱 푸시.** Anki FSRS 알고리즘을 외부에서 파괴하지 않습니다. 다만, 데이터가 너무 무거워 모바일 전체 동기화가 불가능할 때, 스케줄러 에이전트가 "오늘 복습할 카드"만 뽑아 작은 미니 덱(Mini Deck)을 모바일로 푸시합니다.

### [Phase 6] 무제한 스냅샷 백업 시스템
* 에이전트가 Anki DB나 마크다운을 수정하기 전, 구글 드라이브(5TB)에 무조건 `YYYYMMDD_HHMMSS.zip` 형태로 폴더 전체를 백업해 무한 롤백을 보장합니다.

## 3. 카드 생성 규칙 및 트리거 (Trigger Rules)

마진노트 노드 코멘트 첫 줄에 아래 키워드를 입력하면, 엔진이 알아서 Anki 카드의 형태를 변환합니다. 텍스트, 이미지, 혼합 형태 모두 지원됩니다.

| 트리거 키워드 | Anki 카드 형태 (앞면 → 뒷면) | 복습 주기 (SRS) |
|-------------|--------------------------|---------------|
| **(없음)** | 제목이 질문 → 코멘트 내용 전체가 답. | 표준 간격 |
| **(정의 자동감지)** | `X이란 Y이다`, `X: Y` 형태 감지 시 X가 가려진 빈칸(Cloze) 카드 추가 자동 생성 | 표준 간격 |
| **`주의:`** | ⚠ [소단원명] 주의할 점은? → 내용 | 짧음 (빠른 반복) |
| **`강사:`** | 🎤 [소단원명] 강사 보충 설명 → 내용 | 표준 간격 |
| **`절차:`** | 앞 단계를 보여주고 다음 단계를 맞추는 점층적 1~4단계 카드 무더기 생성 | 짧음 (단기 체화) |
| **`공식:`** | WACC = E/V ... 식에서 각 요소를 하나씩 뚫어놓은 빈칸 카드 생성 | 짧음 (단기 체화) |
| **`가림:`** | 텍스트 없이 표/재무제표 이미지가 있을 때 빨간 사각형으로 가려놓고 맞추는 Image Occlusion | 표준 간격 |
| **`분개:`** | [상황 텍스트] 분개는? → 차/대변 답안 (또는 이미지) | 짧음 (단기 체화) |
| **`빈출`** | 카드 생성 안 함. 태그만 달아둠. (✨ 개선: 최대 주기를 꺾어버리는 방식 폐기, 시험 D-30에 Filtered Deck으로 강제 복습) | FSRS 정규 스케줄 |

## 4. 메타인지 추적 및 약점 연결망 (2차 대응 시스템)

이 시스템의 정수는 "수동으로 틀린 이유를 적지 않고, 구조적인 연결만으로 통계를 뽑아내는 것"입니다.

### 4.1. 오답 원인망 형성 (Hermes 자동 링킹 시스템)
* 과거에는 마인드맵에서 반대편 끝 노드까지 선을 긋거나(Spaghetti Links), 고유 URL을 귀찮게 복사해야 했습니다.
* **현재 방식:** 2차 연습서를 풀다 '자산손상'을 몰라서 틀렸다면, 헤르메스 에이전트(Slack/Telegram)에게 **"헤르메스, 1.3.2 문제(연습서)랑 1.1.1(자산손상 목차) 연결해"**라고만 칩니다.
* 에이전트는 Phase 0에서 만들어진 인덱스 DB를 조회해 두 노드의 URL을 찾아 백그라운드에서 하드 링크(Hard Link)를 생성합니다. 마크다운에는 `틀린이유: [[8장 자산손상]]` 태그가 자동으로 박힙니다.

### 4.2. 궁극의 오답 통계 (Anki + Obsidian 대시보드)
* **진실의 방 (Anki):** 사용자가 Anki 앱에서 `[다시(Again)]` 버튼을 누른 실제 Lapses(망각) 횟수를 DB에서 긁어옵니다.
* **통합 시각화 (Obsidian):** 엔진은 Anki의 오답 횟수와 헤르메스가 그어놓은 원인망(태그)을 취합합니다.
  > 📊 **나의 2차 연습서 약점 랭킹 (실시간 업데이트)**
  > 1. `[[8장 자산손상]]` : 연관 문제 도합 27번 틀림 (치명적 약점)
  > 2. `[[12장 법인세]]` : 연관 문제 도합 15번 틀림

## 5. 요약: 왜 이 그랜드 마스터플랜이 실패하지 않는가?

1. **사용자의 인지적 과부하 최소화:** 모바일에서는 보기만 하고, 수정은 Inbox에 던집니다. 위계는 Logseq 들여쓰기로 직관적으로 짭니다.
2. **환각과 붕괴에 대한 기술적 방어:** 이기종 AI 교차 검증, WebP 자동 압축, SQLite 미러링 복사, 무제한 스냅샷 백업.
3. **메타인지의 극대화:** 단순한 암기 툴을 넘어, 약점 대시보드를 통해 뇌가 무엇을 잊고 있는지 실시간으로 추적하는 **완벽한 CPA 전용 자동화 사이보그 컨설턴트**로 작동합니다.
