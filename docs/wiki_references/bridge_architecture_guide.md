# CPA 초격차 시스템 - 브릿지(TR) 파이프라인 가이드

본 문서는 CPA 초격차 시스템의 모듈 간 데이터를 변환하고 이동시키는 **연결부(Bridge)** 스크립트들의 역할과 구동 원리를 에이전트에게 설명하기 위한 가이드입니다. 

에이전트는 화살표 역할을 하는 `TR_*` 접두어가 붙은 파이프라인 코드를 수정할 때 반드시 이 문서를 참고하여 목적을 훼손하지 않도록 해야 합니다.

## 🌉 브릿지(Transition) 모듈 명명 규칙
`TR_출발번호_도착번호_동사형태.py` 구조를 갖습니다.
예: `TR_1.4_to_2.1_reverse_insert.py`는 [1.4] 모듈의 결과물을 [2.1]로 밀어넣는(역삽입) 역할을 합니다.

## 📋 핵심 브릿지 12개 상세 설명

| 브릿지 파일명 | 설명 (동사형) | 실제 시스템 내의 역할 |
| :--- | :--- | :--- |
| **`TR_1.1_to_1.4_toc_extract.py`** | TOC 추출 및 매핑 | 기본서 PDF의 내장 북마크를 파싱하여 TOC-URL DB 생성을 위한 전처리 수행 |
| **`TR_1.2_to_2.1_mcq_slice.py`** | 문항 단위 슬라이싱 | 1차 문제집을 자잘한 노드 이미지로 쪼개어 마진노트로 임포트하기 위한 OPML 생성 |
| **`TR_1.3_to_2.1_essay_split.py`** | 공통자료/물음 분할 | 2차 연습서를 시나리오와 물음 계층형으로 쪼개어 마진노트로 임포트 준비 |
| **`TR_1.4_to_2.1_reverse_insert.py`** | 뼈대 역삽입 | TOC DB를 바탕으로 빈 마인드맵 구조를 MarginNote에 자동 전개 (Import) |
| **`TR_2.X_to_3.1_db_hook.py`** | SQLite DB 후킹 | 마진노트 앱(2.2 ZCOLOR, 2.3 ZGROUP)에서 데이터가 변경되면 백그라운드 파이프라인(3.1)으로 낚아채는 트리거 |
| **`TR_3.3_to_4.1_logseq_push.py`** | Logseq 푸시 | 분배 엔진(3.3)에서 2차 문제 데이터를 Logseq 계층형 마크다운 포맷으로 변환 및 전송 |
| **`TR_3.3_to_4.2_obsidian_push.py`** | Obsidian 푸시 | 분배 엔진(3.3)에서 개념서 데이터를 Obsidian 위키용 포맷으로 변환 및 전송 |
| **`TR_3.3_to_5.1_anki_push.py`** | Anki 카드 생성 | 분배 엔진(3.3)에서 추출된 문제를 Anki-Connect API를 통해 Anki 스케줄러로 전송 |
| **`TR_5.3_to_6.1_mini_deck_push.py`** | Daily Mini Deck 전송 | Anki에서 모바일 과부하 방지를 위해 "오늘 볼 카드"만 떼어내 모바일로 푸시 |
| **`TR_6.2_to_7.1_inbox_relay.py`** | 오타/에러 수신 릴레이 | 모바일 텔레그램/Slack에서 보낸 텍스트를 Mac 로컬 서버(7.1 봇)로 전달 |
| **`TR_4.2_to_7.2_rag_search.py`** | 지식 RAG 검색 | 튜터 봇(7.2)이 사용자 질문에 답변하기 전, Obsidian(4.2) 문서를 RAG로 검색해 컨텍스트 확보 |
| **`TR_7.2_to_7.3_cross_check.py`** | AI 교차 검증 통신 | 튜터 봇(Claude)과 체커 봇(Qwen/Gemini)이 서로 OpenCode Go 환경에서 논쟁(Cross-check)하며 환각 방지 |

---
**주의사항:** 윈도우(Windows) 에이전트는 코드(브릿지)를 설계 및 수정할 때, 출발지 데이터가 목적지 포맷(예: 마크다운, OPML, JSON, SQLite)에 맞게 안전하게 변환되는지 각별히 유의하십시오.
