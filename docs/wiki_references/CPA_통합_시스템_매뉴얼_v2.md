# CPA 학습 자동화 시스템 — 통합 시스템 매뉴얼 (v2.0)

본 매뉴얼은 단순한 카드 생성기를 넘어, **"내가 무엇을 알고 모르는지(메타인지)"를 완벽하게 통제**하고, 학습 효율을 극대화하기 위한 종합 아키텍처를 정의합니다.

---

## 1. 4대 시스템 아키텍처 및 역할 분담

학습 과정에서 발생하는 모든 데이터는 하나의 원천(MarginNote)에서 출발하여, 각 앱의 특성에 맞게 완벽히 분리 및 통합됩니다.

| 시스템 | 역할 (기능) | 상세 설명 |
|--------|-----------|---------|
| **MarginNote4** | **창고 & 입력단** | 교재, 필기, 오답이 모이는 원천 DB. 사용자가 시각적 마인드맵 선 긋기로 '암묵지'와 '약점'을 메타인지적으로 기록. |
| **Python 스크립트**| **해독 및 분배 엔진** | 마진노트의 SQLite DB를 직접 뜯어내어, 각종 코멘트 트리거와 시각적 선 연결을 해독한 뒤 Anki와 Markdown으로 분배. |
| **Anki** | **자동 복습기 (FSRS)** | 쪼개진 덱 없이 하나의 덱에서 FSRS 망각곡선이 100% 스케줄을 통제. 태그(`단원`, `유형`, `약점`) 기반 그룹화 출제. |
| **Logseq** | **카드 통합 통제실** | 숲의 '나무' 관리. 2차 문제처럼 **[공통 자료]** 하위에 **[물음 1,2,3]**이 달린 다중 위계 카드를 아웃라이너로 제어. |
| **Obsidian** | **메타인지 약점 관제실** | 숲 '전체' 관리. Dataview 및 지식 그래프를 활용해 "내가 어떤 개념 때문에 몇 번 틀렸는지" 실시간 통계 대시보드 제공. |

## 2. 카드 트리거(Trigger) 및 생성 규칙

사용자가 마진노트 노드 코멘트에 아래 트리거를 입력하면, 스크립트가 인식하여 다양한 형태의 Anki 카드를 찍어냅니다.

| 트리거 | 텍스트 입력 예시 | Anki 카드 형태 (앞면 → 뒷면) | 복습 주기 (SRS) |
|--------|----------------|--------------------------|---------------|
| **(없음)** | 발췌나 텍스트 | 제목이 질문 → 내용이 답. (정의 패턴은 빈칸 자동 생성) | 표준 |
| `주의:` | 주의: 내재이자율 우선 | ⚠ [5장] 주의할 점은? → 코멘트 내용 | 짧음 |
| `강사:` | 강사: 실무에선 이거 안 씀 | 🎤 [5장] 강사 보충 설명 → 코멘트 내용 | 표준 |
| `절차:` | 1. ~ 2. ~ 3. ~ | 앞 단계를 보여주고 다음 단계를 맞추는 점층적 카드 3장 | 짧음 |
| `공식:` | WACC = E/V × Re ... | 각 요소를 하나씩 뚫어놓은 빈칸 카드 여러 장 | 짧음 |
| `가림:` | (비교표/T계정 이미지) | Image Occlusion. 특정 사각형 가리고 맞추기 | 표준 |
| `분개:` | (차) 자산 100 / (대) 현금 100 | "분개는?" → 차/대변 답안 | 짧음 |
| `빈출` | 빈출 | 카드 생성 안 함. 해당 노드 산하 카드 전체 최대 주기 단축 | 최대 60일 제한 |

## 3. 오답 관리 및 '약점' 추출 시스템 (가장 중요)

2차 연습서를 풀다가 틀렸을 때, "왜 틀렸는지"에 대한 메타인지 데이터를 1초 만에 시스템에 밀어넣는 방법입니다.

### [사용자 행동: 수동 URL 복사 폐기]
1. 마진노트에서 연습서 오답 노드의 코멘트 첫 줄에 `오답:` 이라고 씁니다.
2. 마인드맵 화면에서, **내가 이 문제를 틀리게 만든 원인 개념 노드**(예: 자산손상, 법인세)를 마우스로 끌어서 연습서 노드에 **점선(Link)으로 툭 이어줍니다.**
   - *끝입니다. 귀찮게 텍스트로 개념을 치거나 URL을 복사할 필요가 없습니다.*

### [스크립트의 자동 처리]
스크립트는 마진노트 DB에 몰래 새겨진 `marginnote4app://` 연결 데이터를 해독하여 다음과 같이 분배합니다.
1. **Anki 카드**: "이 문제는 자산손상 때문에 틀린 오답이다"라고 규정하고, 카드에 `#유형::오답`, `#약점::자산손상`, `#약점::법인세` 태그를 때려 박습니다.
2. **Obsidian 마크다운**: 해당 문제 파일의 백그라운드 데이터(Frontmatter)에 `틀린이유: [[자산손상]], [[법인세]]` 라고 양방향 링크를 영구 기록합니다.

## 4. 2차 문제 '자료(시나리오)'의 다중 위계 통제 (Logseq)

2차 문제는 "방대한 자료 하나"를 읽고 "물음 1, 2, 3"을 풀어야 합니다. Anki에서 이 카드들이 무작위로 흩어지면 맥락을 상실합니다.

1. **Logseq의 통제**: 스크립트가 마크다운을 출력할 때 Logseq의 아웃라이너 위계를 활용합니다.
   ```markdown
   - 📄 2차 연습서 p.127 (공통 자료 이미지)
     - 📝 물음 1 (Anki 카드 1번)
     - 📝 물음 2 (Anki 카드 2번) - #약점::자산손상
     - 📝 물음 3 (Anki 카드 3번)
   ```
2. **복습 동기화**: Anki에서 물음 2번 카드를 풀다가 전체 자료가 필요해지면, 카드 하단의 `[Logseq 열기]` 딥링크 버튼을 눌러 위 화면으로 즉시 순간이동합니다.

## 5. 메타인지 통합 약점 대시보드 (Obsidian)

Anki가 카드를 스케줄링해 준다면, Obsidian은 내 두뇌의 스케줄링을 해줍니다.
스크립트가 자동 기록한 마크다운 데이터를 바탕으로, 옵시디언 Dataview 플러그인이 아래와 같은 **통합 관제 대시보드**를 띄워줍니다.

> **📊 나의 약점 랭킹 (실시간 업데이트)**
> 1위: `[[8장 자산손상]]` - 연관 오답 15회
> 2위: `[[12장 법인세]]` - 연관 오답 8회
> 3위: `[[5장 리스]]` - 연관 오답 3회

사용자는 시험이 다가올 때 이 대시보드를 보고, "아, 내가 개념은 안다고 착각했지만 실제 2차 연습서에 엮일 때 자산손상에서 15번이나 막혔구나"라는 극강의 메타인지를 획득하게 됩니다.

## 6. 시스템 허점 방어망 (Loopholes Mitigations)

1. **카드 중복 방지 (멱등성)**: Anki 카드 내부에 숨겨진 고유 UUID 필드를 삽입하여, 마진노트에서 내용을 수정하고 백 번을 다시 추출해도 카드가 증식하지 않고 **내용만 덮어쓰기(스마트 업데이트)** 됩니다.
2. **태그 파손 방지**: `8.1 자산 손상` 처럼 띄어쓰기가 있는 개념명은 스크립트가 `#약점::8.1_자산_손상` 으로 자동 정제하여 Anki의 태그 트리가 박살나는 것을 방지합니다.
3. **데드 링크 폴백(Fallback)**: 마크다운 폴더 동기화 에러로 Obsidian 링크가 죽더라도, Anki 카드 하단에는 항상 **MarginNote 원문 PDF 링크**가 박혀 있어 언제든 원본 뷰로 살아 돌아갈 수 있습니다.
