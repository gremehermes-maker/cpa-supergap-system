# CPA 학습 자동화 시스템 — 종합 매뉴얼

---

## 시스템 구조

```
MarginNote4     →     공유 Markdown 폴더          →     Anki
(창고)                 ↙      ↓       ↘               (복습)
                Obsidian   Logseq    AI 에이전트       Galaxy에서
                그래프뷰   아웃라이너  LLM Wiki         맞음/틀림만
```

| 도구 | 역할 |
|------|------|
| MarginNote4 | 모든 것의 진입점. PDF 발췌 + 트리거 코멘트 |
| Obsidian | 그래프 뷰 (개념 연결 시각화) + LLM Wiki (AI 지식베이스) |
| Logseq | 아웃라이너 (개별 맥락 탐색) |
| AI (NotebookLM/Hermes) | 공유 폴더의 Markdown을 읽고 Q&A, 분석 |
| Anki | 복습 인터페이스. Galaxy AnkiDroid |

Obsidian과 Logseq는 **같은 폴더**를 읽음. 파일 하나를 두 앱이 다른 뷰로 보여줌.

---

## 마인드맵 = 창고

교재 목차(TOC) 기반 5단계 마인드맵. 모든 것이 여기에 들어감:
- PDF 발췌 (교재 원문)
- 강사 보충 설명
- 강사 필기 사진
- 틀리기 쉬운 포인트
- 절차/순서
- 공식
- 분개
- 오답 분석
- 문제 이미지
- 추가 개념

시간에 따라 계속 쌓임: 강의 후 정리 → 문제 풀면서 추가 → 오답 분석 → 2차 문제 추가.

## 트리거 목록

노드에 코멘트 첫 줄에 트리거를 쓰면 해당 카드가 자동 생성.

| 트리거 | 결과 | SRS |
|--------|------|-----|
| (없음) | 기본 카드 (제목→질문, 내용→답) | 표준 |
| `주의:` | 함정 카드 ("주의할 점은?") | 짧음 |
| `강사:` | 강사 보충 카드 | 표준 |
| `빈출` | 태그만 (노드 전체 카드에 빈출 태그) | 최대60일 |
| `절차:` | 절차 카드 (점층적 빈칸) | 짧음 |
| `공식:` | 공식 카드 (각 요소 빈칸) | 짧음 |
| `가림:` | Image Occlusion | 표준 |
| `분개:` | 분개 카드 ("분개는?") | 짧음 |
| `오답:` | 오답 분석 카드 | 최단 |
| `연결:` | 태그만 (다중 목차 연결, 2차용) | — |

## 트리거별 상세

### 기본 (트리거 없음)
```
노드에 PDF 발췌나 텍스트를 그냥 넣으면 됨.
→ 노드 제목이 질문, 내용이 답
→ "X이란 Y이다" 등 정의 패턴 발견 시 빈칸 카드 추가 생성
```

### 주의:
```
주의: 내재이자율과 증분차입이자율이 모두 주어져도 내재이자율 우선
```
→ 앞면: "⚠ [5.2.1 할인율 결정] 주의할 점은?"
→ 뒷면: 코멘트 내용

### 강사:
```
강사: 실무에서는 거의 증분차입이자율을 쓴다
```
→ 앞면: "🎤 [5.2.1] 강사 보충 설명"
→ 뒷면: 코멘트 내용

### 빈출
```
빈출
```
→ 카드 생성 안 함. 이 노드의 모든 카드에 빈출 태그 + 최대 간격 60일 제한.

### 절차: (점층적 빈칸)
```
절차:
1. 리스료 구성요소 파악
2. 할인율 결정
3. 현재가치 계산
4. 분개 작성
```
→ 4장의 카드가 점층적으로 생성:
```
카드 1: 1. [____]
카드 2: 1. 리스료 구성요소 파악  2. [____]
카드 3: 1. ~  2. ~  3. [____]
카드 4: 1. ~  2. ~  3. ~  4. [____]
```
앞 단계는 보여주고 다음 단계를 맞추는 방식. 순서 자연스럽게 체화.

### 공식:
```
공식: WACC = E/V × Re + D/V × Rd × (1-T)
```
→ 각 요소를 하나씩 가린 카드 N장 생성.

### 가림: (Image Occlusion)
```
가림: (비교표/T계정/재무제표 이미지 첨부)
```
**Image Occlusion이란:**
이미지에 사각형을 그려서 일부를 가리고, 가린 부분을 맞추는 카드.

**작동 방식:**
1. MarginNote4에서 이미지 캡처 (비교표, T계정, 재무제표 등)
2. 시스템이 이미지를 Anki로 전달
3. Anki에서 가릴 부분에 사각형을 그림 (1회만)
4. 사각형 하나 = 카드 1장 자동 생성
5. 복습: 전부 가려져 있고, 하나만 빨간색으로 표시 → 맞춰야 함
6. 뒤집으면 답 공개 → 맞음/틀림

**적합한 것:** 비교표, T계정, 재무제표 양식, 플로우차트, 강사 판서
**부적합:** 긴 텍스트 문단 (위치를 외워버림)

### 분개:
```
분개: 리스 최초 인식
(차) 사용권자산  100
    (대) 리스부채  100
```
→ 앞면: "분개는?" / 뒷면: 분개 내용 (텍스트 or 이미지)

### 오답:
```
오답: 1차 p.52 q7. 증분차입이자율 써서 틀림.
```
→ 앞면: "[5.2.1] 오답 — p.52 q7" / 뒷면: 오답 원인
→ 관련 개념 카드에 약점 태그 자동 추가

### 연결:
```
연결: 8.1, 12.3
```
→ 카드 생성 안 함. 해당 카드에 다중 목차 태그 부여 (2차 시험 대비).

## 넣을 수 있는 형태

모든 트리거에 공통. 텍스트, 이미지, 혼합 전부 가능:

| 넣은 것 | 카드에 들어가는 것 |
|---------|-----------------|
| 텍스트만 | 텍스트 |
| 이미지만 | 이미지 |
| 텍스트 여러 개 | 순서대로 나열 |
| 이미지 여러 개 | 순서대로 나열 |
| 텍스트+이미지 | 텍스트 위, 이미지 아래 |

## SRS 스케줄링

| 카드 유형 | 학습 단계 | 최대 간격 | 설계 의도 |
|----------|----------|----------|----------|
| 기본 (개념) | 1분→10분 | 없음 | 표준 에빙하우스 |
| 함정 | 1분→5분→10분 | 60일 | 자주 확인 |
| 강사 보충 | 1분→10분 | 없음 | 표준 |
| 절차 | 1분→5분→10분 | 60일 | 순서 체화 |
| 공식 | 1분→5분→10분 | 60일 | 공식 체화 |
| 분개 | 1분→5분→10분 | 60일 | 분개 체화 |
| 오답 | 1분→5분→10분→30분 | 30일 | 최단. 집중 복습 |
| IO | 1분→10분 | 없음 | 표준 |

빈출 태그: 해당 노드 전체 카드의 최대 간격 → 60일 제한.

**맞음**: FSRS가 간격 자동 증가.
**틀림**: 간격 리셋 + 관련 카드에 약점 태그 + 오답은 개념 카드도 리셋.

---

## 카드 간 연결 (2차 시험 대비)

2차 문제 = 개념 3~4개 조합. `연결:` 트리거로 다중 태그 부여.

```
오답: 2차 p.127 q2. 손상차손의 법인세 효과 놓침
연결: 8.1, 12.3
```
→ 이 오답 카드에 5.2 + 8.1 + 12.3 태그 모두 부여
→ Logseq에 양방향 링크 자동 생성: [[5.2]] ↔ [[8.1]] ↔ [[12.3]]

1차 SRS ≠ 2차 SRS. 각 카드는 자기 맞음/틀림으로만 스케줄링.

## 맥락 관리 (Obsidian + Logseq + AI)

**Logseq (아웃라이너 — 맥락 탐색):**
```
- 5.2.1 할인율 결정
  - 개념: 내재이자율 우선, 모르면 증분차입이자율
  - 연결
    - [[8.1 자산손상]] → 리스 자산도 손상 대상
    - [[12.3 법인세]] → 감가상각 차이 → [[일시적차이]]
  - 관련 문제
    - [[1차 p.52 q7]] — 오답
    - [[2차 p.127 q2]] — 오답
  - 오답 패턴: 내재이자율 놓치는 실수 (2회)
```

**Obsidian (그래프 뷰 — 전체 연결 시각화):**
개념 노드 간 연결을 그래프로 한눈에 확인.

**AI (LLM Wiki — 지식 베이스):**
NotebookLM Pro, Hermes Agents 등이 같은 Markdown 폴더를 읽고:
- 약점 분석 ("자주 틀리는 패턴이 뭐야?")
- 교차 참조 ("이 개념이 나온 문제 전부 보여줘")
- 학습 전략 제안

**Anki 카드 뒷면에 맥락 링크 포함:**
```
📖 상세 → obsidian://open?... (탭하면 전체 맥락으로 이동)
```

---

## 2차 문제의 자료 처리

2차 문제에는 자료(시나리오)가 있음:
- 앞면 = 자료 + 물음 (이미지)
- 뒷면 = 오답 원인 + 풀이 (이미지)
- 전체 맥락 = Logseq 링크

---

## 문제 풀기 후 마인드맵 업데이트

문제를 풀고 나서 당신이 하는 것:
1. 틀린 문제 → 해당 개념 노드에 `오답:` 코멘트 + 문제 이미지
2. 새 개념 발견 → 해당 노드에 텍스트/이미지 추가
3. 시스템이 추가분을 감지 → 카드 자동 생성

---

## 정의 자동 감지 패턴

PDF 발췌에서 아래 패턴이 발견되면 자동으로 빈칸(Cloze) 카드 추가 생성:
- X이란 Y이다
- X은/는 Y을/를 말한다
- X은/는 Y을/를 의미한다
- X(이)라 함은 Y
- X: Y (콜론 패턴)

## 전체 파이프라인

```
MarginNote4 DB
  → Obsidian Sync 플러그인 → Markdown 생성
  → 공유 폴더에 저장

STT/SRT 파일
  → 강조 패턴 분석 ("중요합니다" 등)
  → 관련 노드에 빈출 태그 자동 부여

공유 폴더
  → Obsidian: 그래프 뷰 + LLM Wiki
  → Logseq: 아웃라이너 + 양방향 링크
  → 카드 생성 엔진: 트리거별 규칙 적용

카드 생성 엔진
  → Markdown 파싱 → 트리거 감지
  → 규칙 기반 변환 (regex, 구조 파싱)
  → AnkiConnect API로 자동 임포트
  → 유형별 SRS 프리셋 적용

Anki 복습
  → Galaxy AnkiDroid에서 맞음/틀림
  → 틀림 → 간격 리셋 + 약점 태그
  → 오답 → 관련 개념 카드도 리셋
```
