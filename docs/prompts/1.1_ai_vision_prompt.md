# 👁️ [모듈 1.1] Gemini Vision 전용 완벽 목차 추출 프롬프트

사용자님께서 새로운 기본서 PDF의 목차 이미지(스크린샷)를 캡처한 뒤, 아래의 프롬프트를 복사하여 제미나이(Gemini) 또는 챗GPT에게 이미지와 함께 붙여넣으시면 완벽한 `_toc.json` 초안을 얻으실 수 있습니다.

---

**[프롬프트 복사 영역 👇]**

당신은 회계사 수험생을 위한 고도화된 문서 구조화(Vision) AI입니다. 
첨부된 책의 목차(Table of Contents) 스크린샷 이미지들을 주의 깊게 읽고, 아래 규칙에 따라 정확하게 JSON 배열(Array) 형태로 변환해 주십시오.

### 📝 규칙 (Rules)
1. **계층 구조(Hierarchy) 완벽 반영:**
   - 대단원(예: 1 재무관리의 이해)은 최상위 `title`이 됩니다.
   - 중단원(예: 1 기업과 시장)은 그 아래의 `children` 배열로 들어갑니다.
   - 소단원(예: 1. 기업의 다양한 활동, (1) 조달) 등은 계속해서 하위 `children`으로 깊숙이 들어갑니다.
   - 가장 하위 노드의 `children`은 빈 배열 `[]`로 남겨둡니다.

2. **텍스트 형식 통일:**
   - 이미지에 있는 글씨, 숫자, 괄호 기호 등을 그대로 적어주세요. (임의 축약 금지)
   - 제목 앞의 번호 체계(1, 1., (1), ① 등)도 `title` 안에 그대로 포함시킵니다.

3. **페이지 번호 (Page Number):**
   - 각 목차의 끝부분에 점선(......)과 함께 적혀 있는 숫자를 찾아 `page` 키에 정수(Integer)로 입력해 주세요.
   - 만약 페이지 번호가 지워졌거나 보이지 않는다면, 바로 상위 부모 노드나 이전 노드의 페이지 번호와 동일하게 적어주세요.

4. **출력 형식:**
   - JSON 문법(따옴표, 쉼표, 괄호 짝)에 완벽히 맞아야 하며, 다른 부가적인 설명 없이 오직 JSON 텍스트만 출력해 주세요.
   - 마크다운 코드 블록 ` ```json ... ``` ` 안에 감싸서 출력해 주세요.

### 💻 출력 예시 (Format Example)
```json
[
  {
    "title": "1 재무관리의 이해",
    "page": 4,
    "children": [
      {
        "title": "1 기업과 시장",
        "page": 4,
        "children": [
          {
            "title": "1. 기업의 다양한 활동",
            "page": 4,
            "children": []
          }
        ]
      }
    ]
  }
]
```
