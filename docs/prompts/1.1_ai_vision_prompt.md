# 📝 [모듈 1.1] 텍스트 기반 완벽 목차 추출 프롬프트

자동 추출된 기본서 PDF 텍스트를 바탕으로, 아래 프롬프트를 AI에게 제공하면 완벽한 `_toc.json` 초안을 얻을 수 있습니다.

---

**[프롬프트 복사 영역 👇]**

당신은 회계사 수험생을 위한 고도화된 문서 구조화 AI입니다. 
첨부된 책의 추출 텍스트를 주의 깊게 읽고, 목차(Table of Contents)를 파악하여 아래 지시 사항을 엄격히 따라 주십시오.

### 📝 지시 사항 (Instructions)
1. 들여쓰기나 번호 체계를 주의 깊게 분석하여 정확한 상/하위 계층(Level) 구조를 파악해라.
2. **[중요 검증 단계]** 절대로 처음부터 JSON 코드를 출력하지 마라.
   내가 네가 제대로 분석했는지 한눈에 검증할 수 있도록, 먼저 **마크다운 트리(Markdown Tree)** 형식으로 시각적인 계층 구조를 채팅창에 출력해라.
   *(예시)*
   - 1 재무관리의 이해 (4p)
     - 1. 기업과 시장 (4p)
       - (1) 조달 (4p)
3. 내가 출력된 마크다운 트리를 보고 **"승인"**이라고 말하면, 그때 비로소 해당 구조를 기반으로 아래 규칙에 맞게 완벽한 JSON 코드를 작성해서 출력해라.

---
### 🛠️ "승인" 이후 JSON 변환 규칙
1. **계층 구조:** 대단원은 최상위 객체(title), 중/소단원은 하위 `children` 배열 내부에 위치시킨다. 가장 하위 노드의 `children`은 빈 배열 `[]`로 둔다.
2. **데이터 형식:** 제목 앞의 번호 체계(1., (1), ① 등)를 생략 없이 `title`에 포함시킨다.
3. **페이지 번호:** 목차 우측 끝의 숫자를 찾아 `page` 키에 정수(Integer)로 입력한다. 식별 불가 시 부모 노드의 페이지를 따른다.
4. **포맷:** 결과물은 반드시 마크다운 ` ```json ... ``` ` 블록으로 감싸서 출력한다.
