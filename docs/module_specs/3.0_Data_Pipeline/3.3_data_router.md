# 3.3 데이터 분배 엔진 (3.3_data_router.py)

## 1. 개요
2.0 단계에서 정제된 마진노트 데이터를 목적지(Logseq, Obsidian, Anki)의 포맷에 맞게 변환하여 푸시(Push)하는 핵심 배관공 스크립트입니다.

## 2. 시스템 안정성 및 방어 로직
- **AnkiConnect 포트 침묵 실패 방어 (Mitigation 9.1.4 - Health Check):**
  동기화 시작 전 `http://localhost:8765` 로 핑(Ping) 테스트를 수행합니다. Anki가 꺼져있을 경우 `open -a Anki` 로 강제 실행하며, 그래도 무응답 시 Slack 알람을 발생시켜 '에러 없이 데이터가 누락되는 현상'을 방지합니다.
- **분산 DB 간 시점 불일치 해결 (Mitigation 9.2.2 - Atomic Backup):**
  데이터 분배 시, 1) 마진노트 `.marginnotes`, 2) Obsidian Vault 마크다운 폴더, 3) Anki `collection.anki2` 파일을 한 날 한 시에 하나의 단일 `.zip` 파일로 묶어 백업합니다. 롤백 시 이 세 곳을 동시에 덮어씌워 고아 데이터(Orphan Data) 발생을 원천 차단합니다.
