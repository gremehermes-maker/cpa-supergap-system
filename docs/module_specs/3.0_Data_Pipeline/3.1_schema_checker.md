# 3.1 스키마 무결성 검사 (3.1_schema_checker.py)

## 1. 개요
마진노트 앱 메이저 업데이트 등으로 인한 DB 구조 변경("침묵 실패")을 방어하는 최전선 파이프라인 모듈입니다.

## 2. 비공식 DB 파싱 취약성 방어 (Mitigation 9.1.2)
- **위험성:** MarginNote 메이저 업데이트 시 내부 `.marginnotes` SQLite 테이블 스키마가 약간만 변경되어도 추출 스크립트 전반에 치명적 에러가 발생할 위험이 큽니다.
- **해결책 (Sanity Check):** 파이프라인 실행 시 가장 먼저 `PRAGMA table_info(ZBOOKNOTE)` 명령을 실행하여 스키마를 검사합니다. `ZNOTEID`, `ZCOMMENTTEXT` 등 핵심 컬럼의 존재 여부를 검증하고, 변경이 감지되면 즉각 파이프라인을 멈추고 Slack/Telegram 봇으로 **"🚨 [긴급] MarginNote DB 스키마 변경 감지! 동기화를 강제 중단합니다"** 라는 강력한 알람을 송출합니다.
