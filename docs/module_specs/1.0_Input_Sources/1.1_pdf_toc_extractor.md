# 1.1 기본서 PDF 목차 추출 (하이브리드 워크플로우: AI 추출 + GUI 튜닝)

## 1. 개요 및 설계 철학
이 모듈은 과거의 복잡한 정규식 폰트 파싱을 폐기하고, **"AI 비전 시스템의 초벌 추출 + 인간의 Python GUI 시각적 튜닝"**을 결합한 하이브리드 워크플로우로 진화했습니다. 

AI가 사람처럼 책을 훑고 초기 JSON 트리를 구성하면, 사용자는 시각화된 파이썬 GUI 에디터에서 드래그 앤 드롭 수준의 직관적인 조작으로 계층을 다듬고 최종 결과물을 OPML로 확정합니다. 이를 통해 수십 권의 교재를 마진노트에 넣고 빼며 테스트하는 헛수고를 방지합니다.

## 2. 하이브리드 파이프라인 작동 원리

### Phase 1: AI 초벌 추출 (Antigravity 프롬프트)
사용자는 Antigravity(AI 에이전트)에게 표준 프롬프트를 사용하여 1차 추출을 지시합니다.
```text
[PDF 목차 초안 추출 지시서]
타겟 파일: (파일 경로 지정)
지시 사항: 너의 Vision 능력을 사용해서 목차의 상하위 계층(Level) 구조를 파악하고, `_toc.json` 파일로 초안을 생성해라.
```

### Phase 2: Python GUI 시각적 튜닝 (`1.1_pdf_toc_extractor.py`)
AI가 만들어준 `_toc.json` 초안을 불러와서, 윈도우 폴더 형식(Treeview)으로 시각화하는 파이썬 GUI 프로그램입니다.
*   **핵심 기능:**
    *   **JSON 불러오기:** AI가 생성한 JSON 초안 로드
    *   **계층 구조 변경:** `들여쓰기(Indent)`, `내어쓰기(Outdent)` 버튼을 통한 Level 조정
    *   **노드 텍스트 수정:** 오타나 잘못 인식된 이미지 번호 수정
    *   **노드 추가/삭제:** 누락된 단원 추가 및 불필요한 단원 삭제
    *   **최종 저장:** 튜닝이 완료된 트리를 덮어쓰기 저장(`_toc.json`) 및 마진노트용 `_toc.opml`로 최종 출력

## 3. 입출력 스펙
- **입력:** 기본서 원본 PDF 파일 (AI 추출용), `_toc.json` 초안 (GUI 에디터용)
- **출력:** 최종 검증된 계층화된 `_toc.json` 및 `_toc.opml`
- **연계:** 추출된 최종 목차는 이후 `1.4_toc_url_mapper.py` 모듈 연계를 위해 사용됩니다.
