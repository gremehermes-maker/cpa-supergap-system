# 1.1 기본서 PDF 목차 추출 및 강제 주입 시스템 (Hybrid Workflow)

## 1. 개요 및 설계 철학
이 모듈은 과거의 복잡한 정규식 폰트 파싱을 폐기하고, **"AI 비전 시스템의 초벌 추출 + 인간의 시각적 튜닝 + PDF 원본 자체에 북마크(목차)를 강제 주입(Injection)"**하는 3단계 하이브리드 워크플로우로 최종 진화했습니다. 

마진노트4(MarginNote 4)가 외부 마인드맵(OPML 등) 가져오기를 차단하고 "PDF 원본의 내장 목차를 읽어들여 발췌 시 자동으로 그룹화(Group by Document Outline)" 하는 방식으로 워크플로우를 강제함에 따라, 파이썬(PyMuPDF)을 이용해 PDF 파일 자체의 심장부에 우리가 원하는 완벽한 목차를 강제로 꽂아 넣는 초격차 우회 솔루션을 도입했습니다.

## 2. 3단계 하이브리드 파이프라인 작동 원리

### Phase 1: AI 목차 초벌 추출 (Gemini Vision)
사용자는 기본서 PDF의 목차(Index) 페이지만 캡처한 뒤, 사전에 치밀하게 설계된 전용 프롬프트(`ai_vision_perfect_toc.md`)를 AI에게 제공합니다.
AI는 이미지와 프롬프트 규칙을 기반으로 오차 없이 구조화된 JSON 데이터(`_toc.json`) 초안을 즉시 뽑아냅니다.

### Phase 2: Python GUI 시각적 튜닝 (`1.1_pdf_toc_extractor.py`)
AI가 만들어준 `_toc.json` 초안을 불러와서, 윈도우 폴더 형식(Treeview)으로 시각화하는 파이썬 GUI 프로그램입니다.
* **핵심 조작:** 마우스 드래그 앤 드롭(Drag & Drop)을 통한 직관적인 노드 상하/순서 계층 변경
* **노드 관리:** 추가, 수정, 삭제 기능 지원

### Phase 3: PDF 물리적 북마크 주입 (Injection via PyMuPDF)
사용자가 GUI 튜닝을 마치고 **[최종 저장 및 PDF 북마크 주입]** 버튼을 누르면 다음 작업이 일어납니다.
1. 향후 1.4 모듈(DB 연동)을 위한 `_toc.json` 저장
2. 타겟 원본 PDF 파일을 열고, 파이썬 `PyMuPDF (fitz)` 라이브러리를 통해 방금 튜닝한 완벽한 트리 구조를 PDF의 '공식 북마크(Outline)'로 강제 덮어쓰기
3. `원본이름_목차주입본.pdf` 파일로 새로 생성됨

## 3. 입출력 스펙 및 연계
- **입력:** 기본서 원본 PDF 파일, AI가 추출한 `_toc.json` 초안
- **출력:** 최종 검증된 `_toc.json` (DB용) 및 **북마크가 주입된 `_목차주입본.pdf`**
- **마진노트4 연동:** 주입된 PDF를 마진노트4에서 열고, 형광펜 설정에서 **'문서 목차별 그룹화(Group by Document Outline)'**를 켠 상태로 텍스트를 발췌하면 자동으로 마인드맵 부모 노드가 생성됩니다.
