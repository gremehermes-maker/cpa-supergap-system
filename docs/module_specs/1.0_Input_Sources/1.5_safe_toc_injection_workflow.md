# 재무관리 가이드북 목차(TOC) 주입 프로세스 명세서

본 문서는 '재무관리 가이드북' 작업 시 성공적이고 안전하게 적용되었던 **PDF 단원별 목차 주입 워크플로우**를 상세히 기록한 문서입니다. 시스템 환경을 훼손하지 않고 순수 데이터 가공에 집중했던 가장 모범적인 작업 형태를 보존하기 위해 작성되었습니다.

---

## 1. 작업 개요 (Overview)
- **목표**: 이미 챕터별로 분할된 PDF 파일에, 별도로 준비된 계층형 JSON 데이터를 파싱하여 올바른 페이지 오프셋(Offset)이 적용된 북마크(TOC)를 삽입합니다.
- **도구**: Python 3.x, `PyMuPDF (fitz)`, `json` 라이브러리
- **안전 원칙**: 
  1. 원본 PDF 및 원본 JSON 파일은 절대 수정 및 삭제하지 않음 (비파괴적 작업).
  2. 작업용 스크립트는 시스템 환경과 분리된 임시 폴더(`scratch/`)에서만 작성 및 실행.
  3. 결과물은 반드시 `_목차주입본.pdf` 형태의 새로운 파일로 생성.

---

## 2. 사전 준비 및 구조 분석

작업 전, 주입할 대상의 데이터 구조와 페이지 차이를 분석합니다.

### 2.1. JSON 데이터 구조 파악
미리 정의된 `chapX_toc.json` 파일은 다음과 같은 계층형(Tree) 구조를 가집니다.
```json
[
  {
    "title": "2. 자본예산",
    "page": 25,
    "children": [
      {
        "title": "1. 회수기간(Payback Period)법",
        "page": 25,
        "children": []
      }
    ]
  }
]
```

### 2.2. 페이지 오프셋(Page Offset) 계산
> [!IMPORTANT]
> 원본 교재 기준의 인쇄된 페이지 번호(Logical Page)와, 단원별로 분할된 PDF의 실제 페이지 번호(Physical Page) 간의 차이를 계산하는 것이 가장 핵심입니다.

- **예시**: 챕터 2의 실제 시작 페이지가 원본 교재에서 24페이지라면, 분할된 `Chapter_02.pdf`에서는 24페이지가 1페이지가 됩니다.
- **계산식**: `PAGE_OFFSET = 1 - 원본 교재에서의 시작 페이지 번호` (예: 1 - 24 = `-23`)
- 분할된 PDF의 실제 삽입될 페이지 = `JSON에 적힌 페이지 + PAGE_OFFSET`

## 4. 단계별 실행 및 검증 (Step-by-Step Validation)

> [!TIP]
> 한 번에 모든 챕터를 자동화(Loop)하기 전에, 반드시 단일 챕터에서 선행 테스트를 거쳐야 합니다.

1. **단일 챕터 테스트**: 위 코드를 챕터 2에만 우선 실행하여 예외(Exception) 발생 여부를 확인합니다.
2. **결과물 시각적 확인**: 생성된 `Chapter_02_목차주입본.pdf`를 직접 열어 북마크 트리가 정상적으로 펼쳐지는지, 클릭 시 해당 페이지로 정확히 이동하는지 검증합니다.
3. **루프(Loop) 확장**: 챕터 2의 주입이 완벽함을 확인한 후, 동일한 로직 구조를 챕터 3부터 17까지 배열(Array)과 반복문을 사용해 일괄적으로 안전하게 생성합니다.
