# 6.2 Slack/Telegram Inbox (6.2_telegram_inbox.py)

## 1. 개요
스마트폰 모바일 학습 도중 발견된 오타나 아이디어를 안전하게 기록하는 비동기 로깅(Inbox) 수단입니다.

## 2. 모바일 UX 통제 (Inbox 패턴의 당위성 - 8.4)
- **배경:** 모바일 기기에서의 직접 수정 금지(6.1 원칙)에 따른 보완책입니다.
- **동작 방식:** 폰에서 학습 중 오타를 발견하면, 카드를 수정하지 않고 텔레그램이나 슬랙 봇(7.1 에이전트)에게 `"15번 문제 해설 오타 있음"` 이라는 텍스트만 툭 던져둡니다.
- **처리:** 귀가 후 Mac(로컬) 앞에 앉았을 때 이 Inbox를 확인하여 원본(MarginNote 또는 Markdown)을 직접 수정합니다. 이는 SQLite DB Lock 충돌과 데이터 증발을 막는 가장 강력한 하이브리드 패턴입니다.
