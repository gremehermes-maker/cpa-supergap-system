# 4.2 Obsidian LLM Wiki (4.2_obsidian_wiki.py)

## 1. 개요 (메타인지 관제실)
숲 전체를 조망하고, 약점 통계를 내며, 향후 LLM 에이전트의 지식 베이스(하드디스크) 역할을 하는 관제탑 모듈입니다.

## 2. 궁극의 오답 통계 (진실의 방 대시보드)
- AnkiDroid에서 사용자가 누른 오답 횟수(Lapses)를 취합하고, 헤르메스 봇이 만들어준 '약점 연결망(마크다운 프론트매터)'을 기반으로 Dataview 플러그인과 Graph View를 구동시킵니다.
- 이를 통해 "나의 2차 연습서 약점 랭킹 (예: 1위 자산손상 27번 틀림)"을 실시간 대시보드로 시각화하여 명확한 메타인지를 제공합니다.

## 3. 영구적인 지식 축적 (LLM OS 아키텍처)
- **Persistent Memory (Disk):** 안드레 카파시의 LLM OS 개념에 따라, Memory Editor 에이전트가 세션의 핵심 요약 내용(사용자의 약점, 학습 습관 등)을 이곳 Obsidian의 `LLM_Wiki/User_State.md`에 구조화된 JSON/YAML 형태로 '던져넣어' 데이터를 영구 보존합니다.

구글드라이브로 맥과 윈도우 동시 연동 했을 때의 호환성 문제. 

💡 해결책: 윈도우용과 맥용 설정 폴더를 따로 만들기
옵시디언은 기본적으로 보관소 내의 .obsidian이라는 폴더 하나에서 모든 설정을 불러옵니다. 이 폴더를 기기별로 2개로 나누어 각각 따로 바라보게 만드는 방식입니다.

설정 순서
메인 컴퓨터에서 세팅 완료하기
맥(또는 윈도우) 중 하나를 메인으로 잡고 원하는 플러그인, 테마, 단축키 세팅을 완벽하게 끝냅니다.

설정 폴더 복제하기
메인 컴퓨터의 파일 탐색기(또는 파인더)에서 숨김 폴더인 .obsidian 폴더를 복사하여 그 자리에 2개를 더 만듭니다.

폴더 이름을 각각 .obsidian-mac 과 .obsidian-win 으로 변경합니다.

각 운영체제별로 폴더 지정하기

맥북에서: 옵시디언 설정(Settings) ➔ 정보(About) ➔ 설정 폴더 재지정(Override config folder) 항목에 들어가서 .obsidian-mac 이라고 입력하고 재시작합니다.

윈도우에서: 옵시디언을 열고 동일하게 설정 ➔ 정보 ➔ 설정 폴더 재지정에 들어가서 .obsidian-win 이라고 입력하고 재시작합니다.

이 방식을 썼을 때의 결과
장점: 마크다운 텍스트와 PDF 파일 자체는 구글 드라이브를 통해 완벽하게 양쪽으로 공유되면서, 단축키나 화면 레이아웃, 폰트 등 환경 설정은 철저하게 윈도우와 맥이 독립적으로 작동합니다. 충돌이 발생할 일이 0%가 됩니다.

단점: 새로운 플러그인을 설치하거나 테마를 변경하고 싶을 때, 윈도우에서 한 번, 맥에서 한 번 총 두 번씩 세팅해 줘야 하는 번거로움이 생깁니다. (하지만 LLM 위키 특성상 초반 세팅만 끝내면 이후에는 플러그인 추가보다 문서 작성에 집중하게 되므로 크게 불편하지 않습니다.)
